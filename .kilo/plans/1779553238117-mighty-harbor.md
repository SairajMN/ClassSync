# Fix LayoutGroupContext.mjs.map 404 Errors

## Problem
Browser console shows repeated 404 errors for `LayoutGroupContext.mjs.map` when using framer-motion with Next.js. This is a known issue where webpack-generated source maps reference files that don't exist in the build output.

## Root Cause
- Framer Motion includes source maps inline but webpack configuration doesn't extract `.map` files for node_modules
- DevTools automatically requests `.map` files when they're referenced in JS files

## Solution Options

### Option 1: Disable source maps for production (Recommended)
Add to `next.config.mjs`:
```js
const nextConfig = {
  productionBrowserSourceMaps: false,
  webpack: (config, { dev, isServer }) => {
    if (!dev) {
      config.devtool = false;
    }
    return config;
  },
};
```

### Option 2: Generate source maps properly
```js
const nextConfig = {
  webpack: (config, { dev, isServer }) => {
    if (dev) {
      config.devtool = 'eval-source-map';
    }
    config.output.sourceMapFilename = '[file].map';
    return config;
  },
};
```

### Option 3: Ignore specific source map requests
Add to `next.config.mjs`:
```js
async headers() {
  return [
    {
      source: '/:all*(*.map)',
      headers: [{ key: 'Cache-Control', value: 'no-store' }],
    },
  ];
}
```

## Recommended Fix
Option 1 is the cleanest - disable production browser source maps since they're not essential for the app to function. The 404 errors are harmless but noisy in the console.

## Implementation Steps
1. Edit `next.config.mjs` to disable source maps
2. Clean `.next` directory and restart dev server
3. Verify errors are gone
